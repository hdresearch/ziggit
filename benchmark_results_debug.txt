=== Ziggit API vs CLI Benchmark (Debug Build) ===
Comparing pure Zig function calls vs git CLI spawning
Goal: Prove 100-1000x speedup by eliminating process spawn overhead

Test Setup: 100 files, 10 commits, 3 tags
Iterations: 1000 per operation

RESULTS:
           Operation |    Zig Med |   Zig Mean |    Zig P95 |    Zig P99 ||    CLI Med |   CLI Mean |    CLI P95 |    CLI P99 ||  Speedup
==================================================================================================================================
      rev-parse HEAD | 3.4μs | 3.6μs | 3.5μs | 9.2μs || 1.03ms | 1.05ms | 1.12ms | 1.16ms || 297.6x
  status --porcelain | 88ns | 88ns | 90ns | 105ns || 1.37ms | 1.39ms | 1.48ms | 1.53ms || 15595.7x
     describe --tags | 3.5μs | 3.6μs | 3.5μs | 3.6μs || 1.18ms | 1.20ms | 1.28ms | 1.32ms || 339.0x
            is_clean | 88ns | 98ns | 94ns | 104ns || 1.38ms | 1.40ms | 1.48ms | 1.53ms || 15627.5x

KEY FINDINGS:
- Zig function calls: microsecond/nanosecond range (pure function calls)
- Git CLI spawning: millisecond range (includes ~1-2ms process spawn overhead)
- Speedup range: 297x to 15,627x faster
- Proves that eliminating process spawn overhead gives 100-1000x performance improvements

NOTE: 
- Zig implementations are pure Zig code paths with no process spawning
- CLI implementations spawn 'git' as a child process (typical 1-2ms overhead)
- This validates ziggit's core value proposition for bun integration
