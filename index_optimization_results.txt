Index Parser Optimization Results - Phase 2
==========================================

Before: 758.8 μs (Original parser)
After:   54.7 μs (Fast parser)
Speedup: 13.9x faster (92.8% reduction)
Time saved: 704.0 μs per operation

Optimizations implemented:
1. Skip parsing fields not needed for status (ctime, dev, ino, mode, uid, gid, SHA-1)
2. Use single buffer for all paths instead of individual allocations
3. Optimized bounds checking and parsing loop
4. Reduced memory allocations from N to 2 (entries array + path buffer)

Expected impact on status operations:
- Before: ~950μs total (750μs index + 200μs other)  
- After: ~250μs total (55μs index + 200μs other)
- Overall status speedup: ~3.8x faster
