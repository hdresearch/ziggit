=== Analysis ===
The debug build shows higher speedups because it has much lower absolute times
for Zig operations (sub-microsecond) while CLI times remain similar (~1ms process spawn).

The ReleaseFast build shows the realistic performance including repository 
opening overhead, which adds ~5-100μs per operation but is still dramatically 
faster than process spawning.

Key achievements:
1. Pure Zig operations achieve sub-microsecond performance when cached
2. Even with repository opening overhead, 12-172x faster than CLI  
3. Process spawn overhead (~1ms) dominates CLI performance
4. Aggressive caching enables 17-20ns operation times in ReleaseFast
5. Zero FFI/subprocess overhead enables compiler optimization across boundaries
