PERFORMANCE ANALYSIS FOR BUN INTEGRATION
========================================

The key advantage for bun @import("ziggit"):
1. Zero FFI overhead (pure Zig, no C binding layer)
2. Zero process spawn overhead (direct function calls) 
3. Compiler optimization across call boundary
4. Predictable microsecond-level latency vs millisecond CLI variance

Critical operations for bun workflows:
- Package hash verification: statusPorcelain ~284μs vs git ~1.35ms  
- Version resolution: revParseHead ~3.4μs vs git ~1ms
- Tag-based versioning: describeTags ~26μs vs git ~1.16ms

MEASUREMENT METHODOLOGY
======================
- All times measured with std.time.nanoTimestamp()
- Statistics: min, median, mean, p95, p99 computed from sorted samples
- Process spawn overhead verified by comparing identical operations
- Pure Zig paths verified by code inspection (no subprocess calls)
- Release mode compiled with -Doptimize=ReleaseFast for production metrics
