API vs CLI Benchmark Results - Phase 1 (Debug Build)
======================================================

Test Setup:
- Repository: /tmp/ziggit_bench_repo  
- 100 files, 10 commits, 3 tags
- 1000 iterations per operation

Results (all times are wall clock, mean values):

1. rev-parse HEAD:
   - Zig API: 6.2 μs  
   - Git CLI: 1.03 ms
   - Speedup: 166.0x faster

2. status --porcelain:
   - Zig API: 949.8 μs
   - Git CLI: 1.39 ms  
   - Speedup: 1.5x faster

3. describe --tags:
   - Zig API: 39.0 μs
   - Git CLI: 1.35 ms
   - Speedup: 34.7x faster  

4. is_clean:
   - Zig API: 949.4 μs
   - Git CLI: 1.39 ms
   - Speedup: 1.5x faster

Key Findings:
- Operations like rev-parse HEAD show massive speedup (166x) due to eliminating process spawn overhead
- Status operations show moderate speedup (1.5x) - room for optimization
- All measured Zig functions are pure Zig code paths with no external process spawning
- Process spawn overhead is ~1ms baseline cost for git CLI
